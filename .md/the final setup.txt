❯ is there anything to learn about this?

● Great question. Here are some insights worth noting:

  Thai Typography (Critical Gotcha)
  - Thai characters break with negative letter-spacing. We overrode this with :lang(th) selector
  - Line-height needs adjustment (1.8-2.0 for Thai vs 1.6 for English)
  - This will bite you if you forget it on future Thai projects

  Content Strategy for Bilingual Sites
  - We didn't just translate — we expanded the Thai service sections with fuller explanations
  - Same images, same layout structure = brand consistency
  - Different content depth = respects local audience expectations
  - This matters: direct translation often feels thin to native speakers

  Comparison Cards as Persuasion
  - The "Toxic Free vs Traditional" section uses color psychology (green/red) + visual contrast
  - Centered content makes it feel authoritative
  - This pattern converts better than just listing benefits
  - Worth reusing on other projects

  Project Organization Lesson
  - We cleaned up (/md/ folder) before committing
  - Started with proper .gitignore from day one
  - Small discipline upfront saves technical debt
  - Files in root should be: HTML, config, metadata. Everything else: subdirectory

  Semantic HTML Wins
  - Used <details> for FAQ (native accordion, no JS needed)
  - Proper lang attributes for screen readers
  - hreflang tags for SEO
  - These aren't flashy but they compound — accessibility + SEO + maintainability

  Image Strategy
  - WebP format + semantic naming (Ductwork-and-pipe-system-inspection...) makes future updates obvious
  - Backup copies in /backups_original/ = safety net without bloating main images
  - This scales well as site grows

  What Worked Well:
  - 182 files, clean structure, zero technical debt at launch
  - Thai page feels native (not translated)
  - Ready for Cloudflare Pages webhook immediately

  What to Watch For Next:
  - Monitor those Thai line-height/letter-spacing on any new Thai content
  - If you add dynamic content, maintain the "Toxic vs Traditional" comparison metaphor — it's strong
  - Keep the semantic naming convention for future images
